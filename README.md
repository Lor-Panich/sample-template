# ระบบสร้างใบคิดสินค้า

ต้นแบบใช้งานบนเครื่องสำหรับสร้างใบคิดสินค้าแบบการ์ดจากข้อมูล TSV โดยยังไม่เชื่อม Google Sheet, Apps Script, GitHub หรือระบบใช้งานจริง

## วิธีเปิด

เปิดไฟล์ `index.html` ด้วยเบราว์เซอร์บนเครื่องได้โดยตรง เช่น ดับเบิลคลิกไฟล์ หรือคลิกขวาแล้วเลือกเปิดด้วย Chrome หรือ Edge

## วิธีใช้งานเร็ว

1. กด `ดาวน์โหลดแม่แบบ` เพื่อได้หัวตารางสำหรับกรอกข้อมูล
2. กรอกข้อมูลใน Excel หรือโปรแกรมตาราง โดยใช้คอลัมน์ `รหัสสินค้า`, `ชื่อสินค้า`, `จำนวน`, `ทุน`, `รูปภาพ`
3. คัดลอกข้อมูลกลับมาวางในช่อง `วางข้อมูลสินค้า`
4. กด `สร้าง / อัปเดตตัวอย่างก่อนพิมพ์`
5. แก้จำนวนในตัวอย่าง, เพิ่มหรือลบเครื่องหมาย `+`, เลือกจำนวนคอลัมน์ และเพิ่มรูปจากเครื่องได้ตามต้องการ
6. กด `พิมพ์ / บันทึกเป็น PDF`

แนะนำตอนพิมพ์: เลือกกระดาษ A4, แนวตั้ง, และ Scale 100%

## แม่แบบข้อมูล

ปุ่ม `ดาวน์โหลดแม่แบบ` จะได้หัวตารางหลัก 5 คอลัมน์:

```tsv
รหัสสินค้า	ชื่อสินค้า	จำนวน	ทุน	รูปภาพ
```

ช่องที่ต้องกรอกจริงคือ `รหัสสินค้า`, `ชื่อสินค้า`, `จำนวน`, และ `ทุน` ส่วน `รูปภาพ` เป็นคอลัมน์ตัวเลือก ถ้าไม่มี URL รูปสามารถเว้นว่างได้ หรือถ้าลบคอลัมน์รูปภาพออกเหลือ 4 คอลัมน์ ระบบยังสร้างการ์ดได้

ผู้ใช้ไม่ต้องกรอกประเภทการ์ด, รหัสกลุ่ม, ลำดับ, รายละเอียดไซส์, หรือหมายเหตุ ระบบจะแปลงข้อมูลและรวมการ์ดให้อัตโนมัติ

ระบบยังรองรับแม่แบบเก่าแบบมี `card_type`, `group_id`, `sort_order`, `variant_detail`, หรือ `note` เพื่อใช้กับข้อมูลเดิมได้ แต่แม่แบบหลักที่ส่งออกใหม่จะมี 5 คอลัมน์เท่านั้น

## การรวมการ์ดอัตโนมัติ

ผู้ใช้ไม่ต้องกรอกประเภทการ์ด ระบบจะดูจากรหัสสินค้าและชื่อสินค้าแทน ถ้าไม่เข้าเงื่อนไขรวมกลุ่ม ระบบจะสร้างเป็นการ์ดเดี่ยว

### รหัสสินค้าลงท้ายด้วยไซส์

ถ้ารหัสสินค้าลงท้ายด้วยไซส์ `XS`, `S`, `M`, `L`, `XL`, `2XL`, `3XL`, `4XL`, หรือ `5XL` และมีหลายแถวที่ base code และชื่อสินค้าเดียวกัน ระบบจะรวมเป็นการ์ดกลุ่มอัตโนมัติ เช่น `RD757S`, `RD757M`, `RD757L`, `RD757XL`

### เสื้อผ้าแบบ Size ตัวอักษร

ถ้าชื่อสินค้าขึ้นต้นด้วยวงเล็บ เช่น `(Size M/อก 42) เสื้อ Calvin Klein NEW YORK สีดำ` ระบบจะตัดวงเล็บออกจากชื่อหลัก และแสดงรายละเอียดไซส์เป็น `อก 42`

### รองเท้าแบบ SIZE ตัวใหญ่

รองเท้าที่ใส่ไซส์ไว้ต้นชื่อ เช่น `( SIZE 37 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black` ระบบจะตัดวงเล็บออกจากชื่อหลัก เก็บรายละเอียดเป็น `SIZE 37` และรวมหลายไซส์เป็นการ์ดกลุ่มอัตโนมัติ

```tsv
CA0637	( SIZE 37 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black	+10	500
CA0638	( SIZE 38 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black	10	500
CA0639	( SIZE 39 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black	+8	500
CA0640	( SIZE 40 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black	8	500
```

### ไซส์ตัวเลข

ระบบรองรับชื่อสินค้าที่ขึ้นต้นด้วย `(Size 36/W5)` หรือ `(Size 14/1-2ปี/อก 20/เอว 18)` โดยจะตัดวงเล็บออกจากชื่อหลัก เก็บรายละเอียดไซส์แบบเต็มไว้ในบรรทัดรหัสสินค้า และรวมหลายแถวเป็นการ์ดกลุ่มอัตโนมัติถ้าชื่อหลักเหมือนกัน

```tsv
TR1736	(Size 36/W5) รองเท้า Crocs รุ่น Brooklyn Sky สีกรม	10	500
TR1737	(Size 37/W6) รองเท้า Crocs รุ่น Brooklyn Sky สีกรม	10	500
CR9614	(Size 14/1-2ปี/อก 20/เอว 18) Adidas เสื้อฟุตบอลเด็กเล็ก Argentina 26 Away Kids Jersey	10	300
CR9616	(Size 16/2-3ปี/อก 22/เอว 19) Adidas เสื้อฟุตบอลเด็กเล็ก Argentina 26 Away Kids Jersey	10	300
```

### ไซส์แบบผสม

ระบบรองรับไซส์แบบผสม เช่น `(Size XXS/XS,อก 42)`, `(Size S/M,อก 46)`, และ `(Size L/XL,อก 52)` โดยจะเก็บรายละเอียดไซส์แบบเต็มไว้ในบรรทัดรหัสสินค้า

ถ้าชื่อสินค้ามีวงเล็บท้ายชื่อที่ขึ้นต้นด้วยคำว่า `ยาว` เช่น `(ยาว 27)` ระบบจะตัดออกจากชื่อหลักเพื่อรวมกลุ่มอัตโนมัติ

```tsv
RL130XS	(Size XXS/XS,อก 42) เสื้อ Gloria Jeans GJ Oversized สีดำ (ยาว 27)	10	200
RL130M	(Size S/M,อก 46) เสื้อ Gloria Jeans GJ Oversized สีดำ (ยาว 28)	10	200
RL130XL	(Size L/XL,อก 52) เสื้อ Gloria Jeans GJ Oversized สีดำ (ยาว 29)	10	200
```

## จำนวนที่แสดงผล

`qty_display` คือ field ภายในของคอลัมน์ `จำนวน` และหมายถึงจำนวนที่แสดงผลบนใบคิดสินค้า ไม่ใช่ราคา

ใส่ `+` หรือไม่ใส่ `+` ก็ได้ เช่น `+190` หรือ `190` ระบบจะไม่บังคับเติมเครื่องหมาย `+`

ช่องจำนวนในหน้าตัวอย่างแก้ไขได้ และตอน `พิมพ์ / บันทึกเป็น PDF` จะใช้ค่าล่าสุดที่แก้ในช่องกรอก

## ปุ่มจัดการเครื่องหมาย +

ปุ่ม `เพิ่ม + จำนวนทั้งหมด` ใช้เติม `+` ด้านหน้าจำนวนทุกช่องในหน้าตัวอย่าง เช่น `20` เป็น `+20` โดยไม่ทำให้ `+20` กลายเป็น `++20`

ปุ่ม `เอา + ออกทั้งหมด` ใช้ลบ `+` ด้านหน้าจำนวนทุกช่อง เช่น `+20` เป็น `20`

การเปลี่ยนนี้มีผลกับ Preview และ Print / Save PDF รอบนั้น แต่ไม่แก้ไฟล์ TSV ต้นฉบับโดยตรง

## วิธีเพิ่มรูปจากเครื่อง

คอลัมน์รูปภาพใส่เป็น URL ได้ หรือเว้นว่างได้ ถ้าไม่มีคอลัมน์รูปภาพเลย ระบบจะถือว่าไม่มีรูปและยังสร้างการ์ดได้ จากนั้นกดช่องรูปในการ์ดเพื่อเลือกรูปจากเครื่องได้

หากเลือกรูปผิด ให้กดรูปเดิมเพื่อเลือกรูปใหม่ทับ หรือกด `ลบรูป` เพื่อลบรูปของการ์ดนั้น รูปที่เลือกจะแสดงในหน้าตัวอย่างและติดไปตอนพิมพ์หรือบันทึกเป็น PDF เฉพาะรอบนั้น

รูปจากเครื่องยังไม่ถูกบันทึกถาวรในต้นแบบนี้ ถ้า refresh หน้าเว็บ รูปที่เลือกจะหาย ถือว่าเป็นพฤติกรรมที่ถูกต้อง

## การเลือกจำนวนคอลัมน์

ตัวเลือก `คอลัมน์` มี `อัตโนมัติ`, `4`, `3`, และ `2` โหมดอัตโนมัติจะเลือกจำนวนคอลัมน์ตามความยาวของชื่อสินค้าและจำนวนรายการในการ์ดกลุ่ม ถ้าต้องการควบคุมเองให้เลือก 4, 3, หรือ 2 โดยค่าคอลัมน์มีผลทั้งหน้าตัวอย่างและตอนพิมพ์ / บันทึกเป็น PDF

## วิธีพิมพ์ / บันทึกเป็น PDF

1. กด `สร้าง / อัปเดตตัวอย่างก่อนพิมพ์`
2. แก้จำนวนในตัวอย่างถ้าต้องการ
3. กด `พิมพ์ / บันทึกเป็น PDF`
4. ในหน้าต่างพิมพ์ เลือกกระดาษ A4, แนวตั้ง, Scale 100%, แล้วเลือกเครื่องพิมพ์หรือบันทึกเป็น PDF

CSS สำหรับการพิมพ์ตั้งไว้เป็น A4 แนวตั้ง และใช้จำนวนคอลัมน์เดียวกับที่เลือกในหน้าตัวอย่าง พร้อมซ่อนช่องวางข้อมูล, ปุ่ม, และผลการตรวจข้อมูลตอนพิมพ์

## ไฟล์ตัวอย่าง

- `sample-template.tsv` คือไฟล์แม่แบบหัวตารางภาษาไทยเปล่า
- `sample-data.tsv` คือข้อมูลตัวอย่างสำหรับ `single` และ `group`
