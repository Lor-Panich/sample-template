# ระบบสร้างใบคิดสินค้า

ต้นแบบใช้งานบนเครื่องสำหรับสร้างใบคิดสินค้าแบบการ์ดจากข้อมูล TSV โดยยังไม่เชื่อม Google Sheet, Apps Script, GitHub หรือระบบใช้งานจริง

## วิธีเปิด

เปิดไฟล์ `index.html` ด้วยเบราว์เซอร์บนเครื่องได้โดยตรง เช่น ดับเบิลคลิกไฟล์ หรือคลิกขวาแล้วเลือกเปิดด้วย Chrome หรือ Edge

## วิธีส่งออกแม่แบบ TSV

กดปุ่ม `ส่งออกแม่แบบ TSV` เพื่อดาวน์โหลดไฟล์แม่แบบที่มีหัวตารางครบตามรูปแบบนี้:

```tsv
รหัสสินค้า	ชื่อสินค้า	จำนวน	ทุน	รูปภาพ
```

ช่องที่ต้องกรอกจริงคือรหัสสินค้า, ชื่อสินค้า, จำนวน, และทุน ส่วนรูปภาพเป็นคอลัมน์ตัวเลือก ถ้าไม่มี URL รูปสามารถเว้นว่างได้ หรือถ้าลบคอลัมน์รูปภาพออกเหลือ 4 คอลัมน์ ระบบยังสร้างการ์ดได้

ผู้ใช้ไม่ต้องกรอกประเภทการ์ด, รหัสกลุ่ม, ลำดับ, รายละเอียดไซส์, หรือหมายเหตุ ระบบจะแปลงหัวตารางภาษาไทยกลับไปใช้ field ภายในเดิมโดยอัตโนมัติ

ระบบยังรองรับแม่แบบเก่าแบบมี `card_type`, `group_id`, `sort_order`, `variant_detail`, หรือ `note` เพื่อใช้กับข้อมูลเดิมได้ แต่แม่แบบหลักที่ส่งออกใหม่จะมี 5 คอลัมน์เท่านั้น

## วิธีโหลดข้อมูลตัวอย่าง

กดปุ่ม `โหลดข้อมูลตัวอย่าง` เพื่อใส่ข้อมูลตัวอย่างลงในช่อง TSV ทันที ตัวอย่างนี้มีทั้งรหัสเดี่ยวและรหัสไซส์ที่ระบบรวมเป็นการ์ดกลุ่มให้อัตโนมัติ

## วิธีวาง TSV

1. วางข้อมูล TSV ลงในช่องข้อความ
2. กด `สร้าง / อัปเดตตัวอย่างก่อนพิมพ์`
3. ตรวจผลการตรวจข้อมูล
4. แก้ `qty_display` ในการ์ดตัวอย่างได้ก่อนพิมพ์

## การรวมการ์ดอัตโนมัติ

ผู้ใช้ไม่ต้องกรอกประเภทการ์ด ระบบจะดูจากรหัสสินค้าและชื่อสินค้าแทน

ถ้ารหัสสินค้าลงท้ายด้วยไซส์ `XS`, `S`, `M`, `L`, `XL`, `2XL`, `3XL`, `4XL`, หรือ `5XL` และมีหลายแถวที่ base code และชื่อสินค้าเดียวกัน ระบบจะรวมเป็นการ์ดกลุ่มอัตโนมัติ เช่น `RD757S`, `RD757M`, `RD757L`, `RD757XL`

ถ้าไม่เข้าเงื่อนไขรวมกลุ่ม ระบบจะสร้างเป็นการ์ดเดี่ยว

ถ้าชื่อสินค้าขึ้นต้นด้วยวงเล็บ ระบบจะตัดวงเล็บออกจากชื่อสินค้าและใช้เป็นรายละเอียดไซส์ เช่น `(Size M/อก 42) เสื้อ Calvin Klein NEW YORK สีดำ` จะแสดงชื่อสินค้าเป็น `เสื้อ Calvin Klein NEW YORK สีดำ` และรายละเอียดไซส์เป็น `อก 42`

รองเท้าที่ใส่ไซส์ไว้ต้นชื่อ เช่น `( SIZE 37 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black` ระบบจะตัด `( SIZE 37 )` ออกจากชื่อหลัก เก็บรายละเอียดเป็น `SIZE 37` และรวมหลายไซส์เป็นการ์ดกลุ่มเดียวอัตโนมัติ ตัวอย่างเช่น:

```tsv
CA0637	( SIZE 37 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black	+10	500
CA0638	( SIZE 38 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black	10	500
CA0639	( SIZE 39 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black	+8	500
CA0640	( SIZE 40 ) รองเท้า Fitflop lulu adjustable-buckle leather slides black	8	500
```

## ความหมายของ qty_display

`qty_display` คือ field ภายในของคอลัมน์ `จำนวน` และหมายถึงจำนวนที่แสดงผลบนใบคิดสินค้า ไม่ใช่ราคา

ใส่ `+` หรือไม่ใส่ `+` ก็ได้ เช่น `+190` หรือ `190` ระบบจะไม่บังคับเติมเครื่องหมาย `+`

ช่อง `qty_display` ในหน้าตัวอย่างแก้ไขได้ และตอน `พิมพ์ / บันทึกเป็น PDF` จะใช้ค่าล่าสุดที่แก้ในช่องกรอก


## ปุ่มจัดการเครื่องหมาย +

ปุ่ม `เพิ่ม + จำนวนทั้งหมด` ใช้เติม `+` ด้านหน้าจำนวนทุกช่องในหน้าตัวอย่าง เช่น `20` เป็น `+20` โดยไม่ทำให้ `+20` กลายเป็น `++20` ปุ่ม `เอา + ออกทั้งหมด` ใช้ลบ `+` ด้านหน้าจำนวนทุกช่อง เช่น `+20` เป็น `20` การเปลี่ยนนี้มีผลกับ Preview และ Print / Save PDF รอบนั้น แต่ไม่แก้ไฟล์ TSV ต้นฉบับโดยตรง
## วิธีเพิ่มรูปจากเครื่อง

คอลัมน์รูปภาพใส่เป็น URL ได้ หรือเว้นว่างได้ ถ้าไม่มีคอลัมน์รูปภาพเลย ระบบจะถือว่าไม่มีรูปและยังสร้างการ์ดได้ จากนั้นกดช่องรูปในการ์ดเพื่อเลือกรูปจากเครื่องได้ หากเลือกรูปผิด ให้กดรูปเดิมเพื่อเลือกรูปใหม่ทับ หรือกด `ลบรูป` เพื่อลบรูปของการ์ดนั้น รูปที่เลือกจะแสดงในหน้าตัวอย่างและติดไปตอนพิมพ์หรือบันทึกเป็น PDF เฉพาะรอบนั้น

รูปจากเครื่องยังไม่ถูกบันทึกถาวรในต้นแบบนี้ ถ้า refresh หน้าเว็บ รูปที่เลือกจะหาย ถือว่าเป็นพฤติกรรมที่ถูกต้อง


## การเลือกจำนวนคอลัมน์

ตัวเลือก `คอลัมน์` มี `อัตโนมัติ`, `4`, `3`, และ `2` โหมดอัตโนมัติจะเลือกจำนวนคอลัมน์ตามความยาวของชื่อสินค้าและจำนวนรายการในการ์ดกลุ่ม ถ้าต้องการควบคุมเองให้เลือก 4, 3, หรือ 2 โดยค่าคอลัมน์มีผลทั้งหน้าตัวอย่างและตอนพิมพ์ / บันทึกเป็น PDF
## วิธีพิมพ์ / บันทึกเป็น PDF

1. กด `สร้าง / อัปเดตตัวอย่างก่อนพิมพ์`
2. แก้จำนวนในช่อง `qty_display` ถ้าต้องการ
3. กด `พิมพ์ / บันทึกเป็น PDF`
4. ในหน้าต่างพิมพ์ เลือกเครื่องพิมพ์ หรือเลือกบันทึกเป็น PDF

CSS สำหรับการพิมพ์ตั้งไว้เป็น A4 แนวตั้ง และใช้จำนวนคอลัมน์เดียวกับที่เลือกในหน้าตัวอย่าง พร้อมซ่อนช่องวาง TSV, ปุ่ม, และผลการตรวจข้อมูลตอนพิมพ์

## ไฟล์ตัวอย่าง

- `sample-template.tsv` คือไฟล์แม่แบบหัวตารางภาษาไทยเปล่า
- `sample-data.tsv` คือข้อมูลตัวอย่างสำหรับ `single` และ `group`

## ข้อจำกัดของต้นแบบใช้งานบนเครื่อง

- ยังไม่เชื่อม Google Sheet
- ยังไม่ปรับใช้ Apps Script
- ยังไม่ซิงก์หรือส่งขึ้น GitHub
- รูปจาก `image_url` ต้องเป็น URL หรือพาธไฟล์ที่เบราว์เซอร์เปิดได้
- ไม่มีระบบบันทึกข้อมูลถาวรหลังปิดเบราว์เซอร์
